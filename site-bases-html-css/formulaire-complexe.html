Explication

Depuis HTML5, nous disposons de nouvelles catégories d'input: date, email, number, range, tel, url, color. Ils permettent de disposer de "widgets" déjà définis facilitant la saisie pour les utilisateurs. Ses nouveaux inputs s'adaptent aussi à l'outil utilisé pour l'affichage du site. Par exemple, le clavier de certains téléphones portables basculera sur un clavier "pavé numérique" pour un input de type tel.

De même, de nouveaux attributs facilitent la saisie ou la vérification de celle-ci: placeholder permet d'afficher un message à l'intérieur du champ. Celui-ci s'effacera dès que le champ sera saisi. Attention placeholder ne doit pas se substituer à l'élément label. Si vraiment, vous souhaitez supprimer le label pour des raisons esthétiques ,il faut à minima remplacer le label par l'attribut aria-label afin que les lecteurs d'écrans puissent lire le libellé du champ avant d'en demander la saisie. Malgré tout, éviter au maximum la suppression des labels.

En plus de l'attribut required, d'autres attributs permettent d'affiner les controles de saisies

  • min permet, pour les champs date, number ou range, d'indiquer une valeur minimale. Pour les dates, la valeur doit être fournie sous la forme AAAA-MM-JJ
  • max permet de même, pour les champs date, number ou range d'indiquer une valeur maximale.
  • step disponible pour les champs range et number, permet de définir l'incrément du champ c'est à dire la manière dont la valeur évolue (de 1 en 1, de 2 en 2, etc).
  • pattern permet de définir de manière précise le format de saisie attendue via une expression régulière. Nous verrons comment l'utiliser lors du cours javascript.

On peut associer à des zones de texte, d'email, ou d'url, des éléments datalist composé d'éléments option afin que l'internaute dispose de listes pre-renseignées dans lesquelles il pourra faire un choix. Contrairement à une liste déroulante, il pourra aussi saisir une valeur qui n'est pas proposée dans la liste.

Enfin, on peut aussi transmettre un fichier via un input de type file. Lors de la validation, le fichier sera envoyé au programme serveur en même temps que les informations textuelles. Pour que cela fonctionne correctement il faut ajouter, dans la balise form, l'attribut enctype="multipart/form-data".
